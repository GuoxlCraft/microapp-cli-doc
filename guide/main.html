# 主应用

主应用是承载所有子应用的基座，因此我们通过添加一个MicroAppView的组件来完成qiankun的基座搭建。 它具有获取全局路由在分发给各个子路由的功能。所以登录和鉴权也需要在主应用完成，另外字典的获取也是在主应用完成的。

在获取用户权限下的路由数据之后，会在main\src\permission.ts [188-213]进行格式化路由，筛选出子应用的入口路由获取子应用的路由配置，push到 microAppStore 的 applist里。这样我们就得到当前用户权限下所有的子应用的路由及其子路由appList缓存在Vuex中。

# MicroAppView

在微应用路由配置的章节，我们提到在路由懒加载环节，会将微应用入口路由的页面地址导向MicroAppView组件。这个MicroAppView组件，就是所有子应用在主应用基座中展示的视口组件。main\src\components\MicroAppView\index.vue

当路由跳转到子应用的路由的时候，主应用的Router先对其响应。主应用的跳转MicroAppView视口页。首先MicroAppView组件就会根据appList给每一个子应用创建一个容器。

加载视口页时，根据当前路由的路径从appList中筛选出对应的子应用。再调用loadMicroApp这个Api 在对应子应用的容器中注入这个子应用的qiankun实例来完成子应用在主应用中的展示。

# keep-alive 的实现

在通常的qiankun项目中，路由切换时，切出子应用应当销毁当前子应用。但这个子应用内部缓存的keep-alive状态也会跟着丢失。 为了实现全局的可配置 keep-alive。 我们在视口组件中将加载过的子应用的qiankun实例缓存到microList中，在切回当前路由是先遍历microList中是否存在缓存。如果不存在才调用loadMicroApp生成qiankun实例。main\src\components\MicroAppView\index.vue [52-71]

  if (Object.keys(microList).includes(conf.activeRule.toString())) return

  // 缓存当前子应用
  console.log('loading micro app:', conf.name)
  const micro = loadMicroApp(
    {
      ...conf,
      props: { route: appRoutes[conf.name], router: router, action: action, Fetch: fetch }
    },
    {
      sandbox: {
        experimentalStyleIsolation: true
      },
      excludeAssetFilter: (assetUrl: string) => {
        console.log(assetUrl, 'assetUrl')
        return true
      }
    }
  )
  microList[conf.activeRule.toString()] = micro

子应用的路由跳转的实现，是主应用跳转到子应用入口。子应用会根据urlChange响应自身的路由跳转来实现的。也就是说，当当前路由路径不属于这个子应用，这个子应用的路由指向空白。这也就是为什么非活跃的子应用容器没有真实体积的原因。

利用这个原理，我们实现keep-alive。

另外，我们还将主应用获取的是否需要keepAlive的配置，通过tagsViewStore.microCachedViews 广播给子路由。子路由在router-view组件中就可以根据tagsViewStore.microCachedViews的值去判断那些页面需要keep-alive。 main\src\store\modules\tags-view.ts

最后，每次路由跳转时会判断microList中的子应用是否还存在页签，如果没有页签了。就会调用unmount方法将实例卸载，释放内存。main\src\components\MicroAppView\index.vue [81-91]


// 监听关闭的标签、卸载没用的子应用
const unmountMicApp = (newVal: number, oldVal: number) => {
  if (newVal > oldVal) return
  const keys = Object.keys(microList)
  keys.forEach((key: string) => {
    console.log(key, tagsViewStore.microCachedViews, 'key')
    if (!hasCachedViews(key, tagsViewStore.microCachedViews)) {
      microList[key].unmount()
      delete microList[key]
    }
  })
}
