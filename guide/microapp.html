# 微应用路由配置

TIP

首先，本项目是以qiankun.js作为微前端框架的支撑的。所以无论是否开箱即用，都推荐使用者至少了解qiankun.js (opens new window)的使用方式。 了解微前端的一些概念和术语。这样有助于更好的理解本文档。

TIP

本框架的子应用模板，已经支持独立部署。通过(window as any).__POWERED_BY_QIANKUN__来判断当前应用是独立部署还是在qiankun的环境下。如果是qiankun的环境下就需要一些特殊的配置来满足实现。

除了通常的路由配置以外 （参考侧边栏配置的内容）

子应用需要配置入口路由， 以下面的代码为例

TIP

路由配置的数据格式，是参照我公司应用支撑的数据格式返回。在实际开发中对应字段key需要根据实际修改。

首先看子应用入口路由，首先是component属性，当以App:开头时，会在格式化路由的时候识别为微应用入口路由，将相关路由配置的属性注册到qiankun。后面跟着的是对应的子应用开发环境的webpackConfig.port端口号。源码查看main\src\permission.ts [188-213]

function filterMicroAppRoutes(routes) {
  const appList: appConfig[] = []
  const appRoute: { [key: string]: IObj } = {}
  for (const route of routes) {
    if (route.component.startsWith('App')) {
      const appconfig: appConfig = {
        name: route.code,
        entry: '',
        container: `#${route.code}Container`,
        activeRule: '/' + route.url
      }
      console.log(process.env.API_CURENV, 'process.env.API_CURENV')
      if (process.env.API_CURENV === 'base') {
        appconfig.entry =
          location.protocol + '//' + location.hostname + route.component.replace('App', '')
      } else {
        appconfig.entry = location.origin + '/App-' + route.code + '/'
      }
      appList.push(appconfig)
      appRoute[route.code] = route.children
    }
  }
  console.log(appList, 'appList')
  microAppStore.SetAppList(appList)
  microAppStore.SetAppRoute(appRoute)
}

之后，在路由懒加载的配置中同样根据component属性以App:为依据，将入口路由的页面指向MicroAppView组件。源码查看main\src\store\modules\permission.ts [97-137]

// 在这里处理component
if (item.component === '#') {
  item.component = (resolve) => require(['@/layout/index.vue'], resolve)
} else if (item.component === '##') {
  item.component = (resolve) => require(['@/components/ParentView/index.vue'], resolve)
} else if (itemIsMicroApp) {
  item.component = (resolve) => require(['@/components/MicroAppView/index.vue'], resolve)
  item.meta.microComponentName = item.name.toString()
  item.name = 'MicroAppView'
} else {
  const url = item.component.toString()
  item.component = (resolve) => require(['@/' + url], resolve)
}

入口路由的url和code需要统一，它们对应的是子应用名称

TIP

注意，子应用名称是子应用的唯一标识。它既是子应用package.json中的name的值，也是在路由的判断中分辨当前路由是属于哪个路由的依据。
