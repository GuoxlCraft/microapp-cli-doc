# 项目配置

首先子应用有一个唯一的子应用名称作为识别这个子应用的唯一标识。

这里package.json 的name 属性需要改为子应用名称 apps\MicoAppOne\package.json [1-2]

{
  "name": "MicoAppOne",
  "version": "v0.0.1",
  ...
}

为了确保，当前子应用的vue实例挂载在属于这个子应用对应的容器里。 public目前下的index.html 的 vue容器的id 需要改为 ${子应用名称}-app 的格式。 在mian.ts的render方法中，会将vue实例挂载到id为${子应用名称}-app的容器中 apps\MicoAppOne\public\index.html [17-17]

<body>
  <div id="MicoAppOne-app">
  ...

// main.ts
function render(props = {}) {
  const container = (props as any).container
  instance = new Vue({
    router,
    store,
    render: (h) => h(App)
  }).$mount(container ? container.querySelector('#MicoAppOne-app') : '#MicoAppOne-app')
}

# webpack配置

由于qiankun的是通过读取子应用以库的形式导出的入口文件，且要求输出类型为umd(兼容性强)。所以在子应用的webpack的output属性配置时修改 apps\MicoAppOne\build\webpack.base.conf.js [44-66]

 output: {
    library: `${name}-[name]`,
    libraryTarget: 'umd', // 把微应用打包成 umd 库格式
    ...
  }

TIP

注意，qiankun通过import-html-entry引入子应用时，只会读取子应用以库的形式导出的入口文件。所以切记，这里不要做分包（webpack的splitChunks）!!

# 子应用的路由配置

首先子应用模板中，在获取路由的方法中。会先判断当前环境是否在qiankun环境下。如果是，则会通过permissionStore.generateRoutes的方法从sessionStorage中获取路由。 子应用的permissionStore.generateRoutes会根据子应用名称对路由数据做过滤，只留下当前子应用说包含的路由。

TIP

注意，包含在子应用下的路由和子路由，他们的路由名称（这里是code属性）必须以子应用名称开头。这样在主应用和其他应用响应urlChange时，才能分辨当前url是不是属于本子应用的。

 // apps\MicoAppOne\src\permission.ts [34-39]
 if (isMicroApp) {
    permissionStore.generateRoutes().then(() => {
      router.addRoutes(permissionStore.addRouters) // 动态添加可访问路由表
    })
    permissionStore.setIsAddRouters(true)
  }

 // apps\MicoAppOne\src\store\modules\permission.ts [72-94]
  public generateRoutes(): Promise<void> {
    return new Promise((resolve) => {
      // 路由权限控制
      let routerMap: RouteConfig[] = []
      const dataMenus =
        wsCache.get('dataMenus')[0]?.children.find((item) => item.code === appStore.microAppName)
          ?.children || []
      const routes = [
        {
          url: '/',
          component: '#',
          redirect: `/${dataMenus[0]?.url}`,
          code: 'Root',
          noTagsView: true,
          children: dataMenus,
          alwaysShow: false
        }
      ]
      routerMap = getFilterRoutes(routes)
      this.SET_ROUTERS(routerMap)
      resolve()
    })
  }

另外，在qiankun的使用中。可以通过(window as any).__POWERED_BY_QIANKUN__来判断，当前环境是否在qiankun环境下。 但在qiankun环境下时，要注意vue-router的实例中，需要配置base属性为\${子应用名称}\，来进行正确的路由跳转。

// apps\MicoAppOne\src\router\index.ts [119-123]

const createRouter: any = () =>
new Router({
  mode: 'history', // require service support
  base: (window as any).__POWERED_BY_QIANKUN__ ? '/MicoAppOne/' : '/',
  routes: constantRouterMap
})
